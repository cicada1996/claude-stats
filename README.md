# Claude Stats

A live **terminal dashboard** for how many tokens you burn with Claude Code, with
history. Runs entirely in your terminal — no browser, no window to keep open.

Reads Claude Code's own transcript files (`~/.claude/projects/**/*.jsonl`) — nothing
leaves your machine, no API key.

The terminal dashboard and the browser version need **Python 3.9+ and nothing else**
(stdlib only). The native desktop app additionally needs
[`pywebview`](https://pywebview.flowrl.com/) for its window; `app.py` installs it
with `pip install --user pywebview` the first time it can't import it.

## Run

    python3 claudetop.py

or just double-click **start.command** (macOS) — it opens the dashboard in a
Terminal window.

### Keys

| Key | Action |
|-----|--------|
| `7` `3` `9` `0` | history range: 7 / 30 / 90 days / all time |
| `r` | refresh now |
| `q` or `Esc` | quit |

The dashboard refreshes itself every few seconds while it's open.

## What it shows

- **Stat tiles** — tokens today, API-equivalent cost today, all-time tokens, and
  today's cache hit rate.
- **Current 5-hour window** — a fill meter of tokens used vs. your own historical
  peak, with time until it resets and current burn rate.
- **Recent 5-hour windows** — a sparkline of the last windows.
- **Tokens per day** — horizontal bars segmented by model, for the selected range.
- **By model** and **by project** — bars with tokens and API-equivalent cost.

## Activity Monitor — native app (no browser)

A **desktop app** in its own native macOS window (the system WebKit view — no
browser, no localhost, no server). Its focus is the **current 5-hour window**: a
big graph of your cumulative token usage climbing through the window (time on the
x-axis, tokens on the y-axis) with a dashed line for your usual peak, plus a large
"resets in" countdown, live burn rate, and a projection of where you'll land by
reset. Below that: recent windows, today's totals, and a per-model breakdown.

**Install / open:**

- Double-click **`Claude Stats.app`** in your **Applications** folder.
- To (re)build it from this folder's code, double-click **`install-app.command`**
  (or run `./install-app.command`). The bundle carries its own copy of `app.py`,
  `server.py` and `taskman.html`, so it works from Finder without needing access to
  your Documents folder. It installs to `/Applications` when that's writable and
  falls back to `~/Applications` otherwise — the script prints which one it used.
- For development, `python3 app.py` or `run-app.command` runs the same window
  straight from this folder and prints any errors.

The bundle runs against the system `/usr/bin/python3` and its user site-packages —
it does not vendor `pywebview`. If the app bounces without opening a window, check
**`/tmp/claude-stats.log`**, where the bundle sends all its output.

> **Why is the app in /Applications and not here?** macOS blocks apps launched from
> Finder from reading `~/Documents` (a privacy-protected folder). The editable code
> lives here in your code folder; the installed app carries its own copy so it never
> hits that block. Re-run `install-app.command` after you change the code.

### Appearance

The app ships two skins, switchable from the row of buttons at the top. Both draw
the **same data and the same sections** — the 5-hour-window graph is the centrepiece
of each; only the drawing changes.

| Skin | Look |
|------|------|
| `terminal` | the original TUI: monospace character-cell art, the graph plotted in braille dots (`⣿`), frames in box-drawing characters |
| `clean` | plain modern UI: real layout, SVG graph with a filled area and a dashed projection, light + dark following the system |

The `terminal` skin has three palettes:

| Palette | Look |
|---------|------|
| `phosphor` | the original green-on-black CRT glow (default) |
| `amber` | DEC/VT100 amber phosphor |
| `slate` | what a modern terminal emulator actually looks like — low-contrast, no glow |

Your choice is saved to `~/.claude-stats/config.json` and restored on next launch.
(It lives on disk rather than in the page because pywebview clears `localStorage`
on every launch — a theme kept in the browser would silently reset each time.) The
browser version of this view, at `/taskman`, has the same switcher and remembers it
in `localStorage`.

**Icon:** the app icon (a rising trend line on black) is generated by
`make_icon.py` into `AppIcon.icns` and applied by `install-app.command`. To tweak
it, edit the geometry/colors at the top of `make_icon.py` and re-run the installer.
If Finder still shows the old icon after installing, it's just macOS's icon cache —
log out/in, or run `killall Dock Finder`, to refresh it.

## Optional: browser version

If you ever want it in a browser instead, `server.py` serves the same data:

    python3 server.py
    #  dashboard    → http://localhost:8787/
    #  activity     → http://localhost:8787/taskman

All views read the same files; use whichever you like.

## Notes on the numbers

- **Token totals are large because of prompt caching.** Every turn re-reads the whole
  conversation/context, and each re-read is counted again as "cache read" tokens (billed
  at ~10%). The **count** toggle (top bar of the app) switches between four measures,
  each per response summing different parts of the API `usage`:
  - **all** — input + output + cache writes + cache **reads**. The raw number, hugely
    inflated by re-reads; closest to what rate limits actually see.
  - **processed** — input + output + cache writes (no re-reads). The unique tokens the
    model actually handled; still dominated by cache writes (loading files/context).
  - **content** — input + output only. Your prompts plus Claude's replies.
  - **output** — output only. Just what Claude generated.
- **History is saved** to `~/.claude-stats/history.json` — a per-day record (tokens,
  cost, per-model) that accumulates over time and is kept even if Claude Code later
  deletes its transcripts. Past days are frozen once captured; today's row keeps
  updating. It's not shown in the UI yet — it's there for future history views.
- **Cost figures are API-equivalent estimates** — what the usage *would* cost at
  pay-per-token API prices. On a Max plan you pay a flat fee, so treat this as a
  "what am I getting for my money" gauge, not a bill.
- **The 5-hour-window benchmark is your own historical peak**, not Anthropic's plan
  limit (which isn't published as a number). For official plan usage, run `/usage`
  inside Claude Code.

## Pricing

Per-model prices live in the `PRICING` dict at the top of `server.py` — edit them
there if they change; `claudetop.py` uses the same engine.
